# Cosmos SDK 0.53 Upgrade — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump Stride's dependency stack from Cosmos SDK v0.50.14 to v0.53.7 and adapt the codebase to the corresponding API changes across ibc-go, ibc-apps, wasmd/wasmvm, interchain-security, and the Stride-Labs vesting fork, landed as eight per-dependency commits on the `sdk53` branch.

**Architecture:** Bottom-up sweep of the dependency tree. SDK first (it's the root of every other module's imports), then ibc-go (split into three commits: path rewrite → capability removal → API adaptation), then ibc-apps, wasmd, ICS, and finally the vesting fork. Each commit combines the `go.mod` bump with all code adaptations for that dependency. The repo is expected to be non-building between commits and possibly at the end — per the design spec, build/test verification is deferred.

**Tech Stack:** Go 1.23, Cosmos SDK, CometBFT, ibc-go, ibc-apps (PFM / rate-limiting / ibc-hooks), CosmWasm (wasmd + wasmvm), interchain-security, evmos/vesting (Stride-Labs fork).

**Design reference:** `docs/superpowers/specs/2026-04-21-sdk-0.53-upgrade-design.md`.

**Testing note (critical):** Per user direction, test authoring and build verification are deferred for this entire plan. Do NOT write tests, do NOT run `go build`, do NOT run the test suite between tasks. Limit validation to `go mod tidy` after `go.mod` edits (to resolve go.sum and catch gross module-path typos) and `gofmt` on touched files.

**Commit convention:** one commit per task (task 2 has three sub-tasks, each its own commit). Commits are plain `git commit` — no Co-Authored-By trailer for upgrade commits since the user already uses bot PR titles for dep bumps.

## File Structure

This upgrade touches three categories of files:

1. **`go.mod` / `go.sum`** — modified in every task.
2. **`app/app.go`** (≈1860 lines) — modified in almost every task. Concentrates module registration, keeper construction, IBC middleware wiring, module ordering, and the capability scoping that is being removed.
3. **Custom `x/*` modules with IBC wiring** — touched mainly in Task 2a (import path rewrite) and Task 2c (callback signature updates):
   - `x/stakeibc/ibc_middleware.go`
   - `x/staketia/ibc_middleware.go`, `x/staketia/keeper/ibc.go`
   - `x/stakedym/ibc_middleware.go`, `x/stakedym/keeper/ibc.go`
   - `x/records/module_ibc.go`, `x/records/keeper/ibc.go`
   - `x/icaoracle/ibc_middleware.go`
   - `x/icacallbacks/ibc_module.go`
   - `x/autopilot/module_ibc.go`, `x/autopilot/keeper/ibc.go`

Anywhere else referencing the bumped packages is touched mechanically in the path-rewrite sweep (Task 2a and Task 3). Subdirectories under `deps/` (vendored Gaia/Juno for integration tests) are **excluded** from all sweeps — they are third-party copies pinned to their own SDK versions.
